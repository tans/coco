Coco language tooling for Bun

A compact indentation-based language prototype with a tested Lexer 1.0, REPL, CLI, examples, and implementation docs.

Lexer 1.0

The implemented compiler stage converts `.coco` source into a JSON token stream with layout tokens and source locations.

  • Keywords, identifiers, literals, operators, and punctuation
  • Python-style `INDENT` and `DEDENT` block layout
  • Single-line and multi-line comments ignored by design
  • String template detection for parser-time interpolation

Optional runtime

          

            Coco Runtime is separated from the language core. It currently
            provides manifest planning, event routing, scheduling, and plugin
            boundaries without forcing heavy adapters into the compiler.
